# Android Deployment

**Status: current as of Phase 6.1 (2026-07-20).** New document. Describes
the current, actual Android build/verification process. There is no
distribution pipeline yet (no EAS Build configuration, no Play Console
setup) — this document describes what exists today (local builds) and
what's planned, clearly separated.

## Current state: local debug builds only

Android is the only platform this project builds and runs today (see
`system-overview.md`'s "Target platforms"). Verification to date has been
local `gradlew` builds, not a managed build/distribution pipeline.

### Building locally

```powershell
cd apps/mobile
npx expo prebuild --platform android --no-install   # regenerates android/ from app.json
cd android
.\gradlew.bat assembleDebug                          # or assembleRelease
```

This produces `apps/mobile/android/app/build/outputs/apk/debug/app-debug.apk`.
The `android/` directory itself is git-ignored (see
`apps/mobile/.gitignore` and the root `.gitignore`) — it is fully
regenerated by `expo prebuild` from `apps/mobile/app.json` and is never
committed. Any native-project edit that isn't expressible through
`app.json`/a config plugin will be silently lost on the next prebuild;
there are currently no such manual edits in this project.

### App icon, splash screen, and adaptive icon

Configured entirely through `apps/mobile/app.json`, pointing at Branding
Assets v1.0 (`assets/app-icon/`, `assets/branding/logo/`,
`assets/splash/`) — see
[`../branding/branding-v1.md`](../branding/branding-v1.md) for the asset
inventory and [ADR-backed rule](../README.md) that these paths and files
are frozen. `expo prebuild` generates the actual Android
`mipmap-*`/`drawable-*` resources (adaptive icon layers, splash drawable)
from these source assets on every run — verified during Phase 5.6 to
preserve aspect ratio and not introduce distortion.

### What "verification" has meant so far

- `npx tsc --noEmit` (mobile workspace)
- `npx eslint .` (mobile workspace)
- `npx jest` (mobile workspace)
- `./gradlew.bat assembleDebug` — a full native build, run when native
  code, `app.json`, or branding assets change (not required for pure
  JS/TS changes with no native-config implications)
- Manual verification of generated native resources (e.g. inspecting
  `mipmap-anydpi-v26/ic_launcher.xml` and computed aspect ratios) when no
  emulator/device was available in the working environment — this is a
  documented compromise, not a substitute for on-device verification; see
  "Known gaps" below.

## Known gaps (not yet done)

- **No EAS Build configuration** (`eas.json` does not exist anywhere in
  the repo). `tech-stack.md` names EAS Build as the intended mobile
  build/distribution mechanism; it has not been set up.
- **No Play Console / release signing setup.** No release keystore, no
  Play Console listing, no internal-testing track.
- **No CI-driven Android build.** The `gradlew assembleDebug` runs
  recorded to date were all performed manually in a local/session
  environment, not by an automated pipeline. Tracked as part of Phase 6.4.
- **No real-device verification recorded.** All Android verification to
  date has been emulator-absent (build success + generated-resource
  inspection) or, where an emulator/device was available, standard
  debug-build install. No dedicated real-device background/Doze-behavior
  verification of the playback engine has been recorded — see
  [`../architecture/playback-engine.md`](../architecture/playback-engine.md)'s
  Technical Debt section.

## Planned (not started)

EAS Build + Play Console distribution is the intended path per
`tech-stack.md`, to be set up alongside or after Phase 6.4 (CI/CD). No
timeline is committed yet — this is listed here so a future session knows
the intended direction without mistaking intent for completed work.
